# Canonical Terminology

- `unit`: A numbered learning-content work package under `programs/<program-id>/units/<###-slug>/`.
- `brief.md`: Outcome and scope contract for a unit.
- `design.md`: Learning design strategy and constraints.
- `sequence.md`: Dependency-ordered authoring tasks.
- `content-model.md`: Course/Module/Lesson structure and entities.
- `assessment-map.md`: Outcome-to-assessment coverage map.
- `delivery-guide.md`: Facilitation and learner delivery instructions.
- `rubrics/`: Hard-gate quality checks.
- `audit-report.md`: Cross-artifact decision report (`PASS|BLOCK`).
- `outputs/`: Local-first published assets contract.
- `charter.md`: Governance source of truth at `.lcs/memory/charter.md`.

## Legacy Allowlist

Legacy naming (e.g., `specify`, `plan`, `tasks`, `implement`, `constitution`, `speckit_*`) is allowed only in historical docs such as changelog/RFC history sections.
